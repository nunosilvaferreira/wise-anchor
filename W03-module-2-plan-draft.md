### High-level software description

For Module #2, I will expand my existing project called WiseAnchor, a routine
support web app built with JavaScript, React, and Next.js. In Week 02, the app
already allowed the user to view a daily routine, add tasks, edit settings, and
save information in localStorage. For this sprint, I will continue using the
same repository but improve the JavaScript logic so the project clearly fulfills
the JavaScript module requirements.

The software will fulfill the JavaScript module requirements in the following
ways:

1. Display output to the screen:
The app already displays tasks, settings, and calming support information in the
browser. I will add a progress summary and a daily insights area so JavaScript
updates visible results based on user actions.

2. Native Array ES6 functions:
The app already uses array filtering and sorting. I will strengthen this by
adding more JavaScript logic using map, filter, sort, and reduce to calculate
task completion statistics, organize routine sections, and show progress by time
of day.

3. Recursion:
I will add a recursive JavaScript feature to support repeated step-by-step task
guidance, such as breaking a large routine into smaller nested steps or
displaying calming steps in a guided sequence.

4. Use a JavaScript library written by someone else:
I will integrate a JavaScript library such as date-fns to improve date/time
formatting and task scheduling behavior in the app.

Stretch challenge:
I plan to demonstrate throwing and handling exceptions by validating user input
and showing friendly error messages when task data is incomplete or invalid.

By the end of the sprint, WiseAnchor will remain a working interactive app, but
it will also demonstrate deeper JavaScript problem-solving beyond basic web page
interaction. I will update the README to describe the JavaScript concepts used,
publish the updated code in the public GitHub repository, and record a new video
showing both the running software and the code walkthrough.

## Risks and mitigation plan

### Risk 1

I may spend too much time deciding which new JavaScript features to add and lose
time for implementation.

Action plan:
I will keep the scope focused on three core additions only: ES6 progress
calculations, one recursive feature, and one external JavaScript library. I will
avoid adding unrelated features until the required items are complete.

### Risk 2

I may run into bugs while changing existing task logic because the project
already has multiple pages and shared localStorage data.

Action plan:
I will test the app after each small change, keep changes modular, and use one
feature branch or clearly separated commits so I can isolate issues quickly. I
will also keep a simple time log so I can see whether debugging is taking too
much of the sprint.
